📋 Project Overview

Purpose

What is JARVIS?

A fully working real-time voice AI agent. You speak → it listens → it thinks → it speaks back. Complete end-to-end voice conversation with an LLM, no typing required.

Use Case

Demo Booking Assistant

Handles inbound voice queries: schedule product demos, check slot availability, and provide date/time — all via natural spoken conversation.

Interface

Streamlit Web UI

Browser-based control panel with live transcript display, session stats, real-time status indicator (Listening / Thinking / Speaking), and 0.7 s auto-refresh loop.

STT

Groq Whisper

Raw PCM audio from mic is wrapped in WAV and sent to Groq's whisper-large-v3 — the most accurate open STT model, with Groq's sub-200 ms inference speed.

LLM

Groq LLaMA 3.3

llama-3.3-70b-versatile powers all conversation. Full multi-turn chat history maintained in-memory. Replies capped at 256 tokens for natural voice pacing.

TTS

pyttsx3 SAPI5

Windows SAPI5 text-to-speech via pyttsx3 — fully offline, zero API cost, low latency. Voice rate tuned to 165 wpm for natural spoken cadence.

🧩 Component Breakdown — Final State

app.py — Streamlit UI

Responsibilities

  • Module-level _transcript_q (Queue) — thread-safe transcript bridge
  • Module-level _status dict + Lock — thread-safe status bridge
  • _drain_queue() on every rerun — pulls transcript into session_state
  • Start/Stop session orchestration
  • Sidebar: GROQ_API_KEY check, controls, live status indicator
  • Live transcript rendering (st.chat_message format)
  • Stats bar: total turns, user/agent counts
  • 0.7 s auto-refresh via time.sleep + st.rerun()
config.py — Configuration

Responsibilities

  • Load .env via python-dotenv
  • GROQ_API_KEY env var extraction
  • LIVE_MODEL = llama-3.3-70b-versatile
  • INPUT_SAMPLE_RATE = 16 000 Hz
  • CHUNK_SIZE = 1 024 frames (~64 ms)
  • SILENCE_THRESHOLD = 800 RMS (raised from 500)
  • MAX_HISTORY_TURNS = 20 (40 message objects)
  • load_system_prompt() from prompts/system_prompt.txt
core/agent.py — VoiceAgent (complete rewrite)

Responsibilities

  • Groq client init (STT + LLM on same client)
  • Background daemon thread with private asyncio loop
  • _collect_speech(): VAD gate with MIN_SPEECH_FRAMES=5
  • _transcribe(): PCM → WAV → Groq whisper-large-v3
  • _ask_groq(): full messages[] → llama-3.3-70b → reply
  • _speak(): pyttsx3 SAPI5 TTS (offline)
  • _flush_mic_after_speaking(): 400ms + drain — echo killer
  • Thread-safe callbacks to _transcript_q / _status
core/audio.py — AudioHandler

Responsibilities

  • PyAudio I/O stream lifecycle (open/close/terminate)
  • Capture thread: read mic 16kHz → input_queue.put()
  • Playback thread: output_queue.get() → speaker (legacy)
  • Interrupt detection via audioop.rms
  • interrupt(): drain output_queue
  • Thread-safe queue-based producer/consumer
core/memory.py — ConversationMemory

Responsibilities

  • Thread-safe rolling conversation history
  • Turn dataclass: role + text + HH:MM:SS timestamp
  • threading.Lock() protects _turns list
  • Auto-trim to MAX_HISTORY_TURNS × 2 = 40 entries
  • get_history() returns immutable snapshot copy
  • clear() resets between sessions
core/tools.py — Tool Registry

Responsibilities

  • book_demo(name, date, time) → booking ID + confirmation
  • check_slot_availability(date) → 4 available time slots
  • get_current_time() → current datetime
  • TOOL_REGISTRY dict for O(1) dispatch
  • dispatch(name, args) — safe unknown-tool handling

Project started with Groq as the intended LLM backend using mixtral-8x7b-32768. However, the pipeline was entirely a stub — _recv_loop put empty bytes into the output queue, no STT existed, no TTS existed. The app would start but produce no actual voice conversation. Status stayed at "Connecting" permanently.

Phase 2 — First Migration Attempt (Failed)
Gemini API (google-generativeai) — Deprecated SDK
Switched to Gemini API using the google-generativeai package. Immediately hit a FutureWarning: the entire package was end-of-life. Google had deprecated it in favour of google-genai. Also attempted using Gemini's audio transcription for STT, but hit Forbidden (403) from Google's Web Speech API — geographically restricted in the development environment.

Gemini API (google-genai) — Model Deprecated
Migrated to the new google-genai SDK with updated API calls. Switched model to gemini-2.0-flash. Hit immediate 404 NOT_FOUND: the model was no longer available to new users. Tried gemini-2.5-flash next — same result or unknown model naming instability. Google's rapid model deprecation cycle made it unreliable as a foundation for a real project.

🐛 Critical Bug Fixes — Root Cause Analysis

BUG 1 — FIXED

Streamlit Thread Isolation Violation

Symptom: AttributeError: st.session_state has no attribute "transcript" — agent crashed on every transcription.

Root cause: Streamlit's session_state is bound to the main script execution context (ScriptRunContext). Writing to it from a background daemon thread is undefined behavior — it raises KeyError or AttributeError.

Fix: Introduced module-level _transcript_q (queue.Queue) and _status (dict + threading.Lock). Agent writes to these — safe from any thread. Streamlit reads from them on every rerun via _drain_queue().

BUG 2 — FIXED

Acoustic Echo Loop

Symptom: JARVIS was speaking continuously without listening — it was having a conversation with itself.

Root cause: PyAudio's capture thread runs at all times, including while pyttsx3 is playing TTS through the speaker. The speaker audio was picked up by the microphone, fed back into input_queue, transcribed by Whisper, and replied to — creating an infinite self-conversation loop.

Fix: _flush_mic_after_speaking() — immediately after _speak() returns: sleep 400ms for echo to decay, then drain all accumulated chunks from input_queue before the next listen cycle begins.

BUG 3 — FIXED

Noise Hallucination / Phantom Input

Symptom: JARVIS would start speaking without the user saying anything — answering questions nobody asked.

Root cause: SILENCE_THRESHOLD=500 sat below Windows ambient noise floor (fan, AC hum, electrical noise typically 300–700 RMS). A brief noise spike triggered speech_started=True with just 1 frame, then 1.6s of near-silence was sent to Whisper — which hallucinated text ("Thank you.", "You", random phrases) from the noise.

Fix: Raised SILENCE_THRESHOLD to 800 RMS. Added MIN_SPEECH_FRAMES=5 gate — requires at least 5 consecutive frames (≈320ms) above threshold before treating input as real speech. Noise bursts are silently discarded.

🚀 What We Built

A fully working, production-quality real-time voice AI agent built entirely in Python — from raw microphone PCM to spoken AI response — using three battle-tested technologies.

You speak into your microphone. JARVIS transcribes your words using Groq's Whisper, reasons about them using LLaMA 3.3 70B, and speaks the reply through your speakers using Windows SAPI5 — all in a seamless, echo-free, multi-turn conversation loop.

🎙️

Real-Time Voice Pipeline

End-to-end: microphone → VAD → Groq Whisper STT → Groq LLaMA LLM → pyttsx3 TTS → speaker. Each step runs asynchronously in a background thread, never blocking the UI.

🧠

Context-Aware Conversation

Full multi-turn chat history maintained across the entire session. JARVIS remembers what you said 10 turns ago. History is bounded to 40 messages and trimmed automatically — no context window overflow.

🔇

Echo-Free Audio

Post-TTS mic flush (400ms decay + queue drain) ensures JARVIS never hears itself. VAD noise gate (800 RMS + 320ms minimum) blocks hallucinations from ambient noise. Clean conversation every turn.

🧵

Thread-Safe Architecture

Three threads (Streamlit main, agent asyncio, PyAudio capture) communicate exclusively through queue.Queue and threading.Lock — zero race conditions, zero cross-thread session_state violations.

🛠️

3 Built-In Voice Tools

Book a product demo by voice. Check availability on any date. Get the current time. All wired to Groq's function-calling schema and dispatched in real time during conversation.

📊

Live Web Dashboard

Streamlit UI shows the full conversation transcript, real-time status (Listening / Thinking / Speaking), and session stats — all updating every 0.7s without page reload.
