­¤ōŗ Project Overview ŌĆö Every Detail

Purpose

What is JARVIS?

A fully working real-time voice AI agent. Speak ŌåÆ mic captures 16 kHz PCM ŌåÆ VAD detects Ōēź320 ms of real speech (RMS > 800, Ōēź5 frames) ŌåÆ Groq Whisper transcribes ŌåÆ Groq LLaMA reasons ŌåÆ pyttsx3 SAPI5 speaks back ŌåÆ echo flush clears mic ŌåÆ next turn. Complete hands-free voice loop. No typing required.

Use Case

Demo Booking Assistant

Handles inbound voice queries: schedule product demos (asks for name + date if not given), check slot availability on any date (4 fixed slots), report current time. Persona + 6 behavioral rules live in prompts/system_prompt.txt. Every reply Ōēż 3 sentences ŌĆö optimised for speech, not screens.

Platform & Repo

Windows 11 / Python 3.12

Developed and runs on Windows 11 (build 10.0.26200). Python 3.12 runtime. Virtual env: voiceagent (Miniconda / venv). TTS via Windows SAPI5. Repo: HaRsH00000007/Voice_agent ┬Ę branch main ┬Ę 3 commits. PyAudio wheel: pipwin install pyaudio if pip fails.

Interface

Streamlit Web UI

Sidebar: API key validation badge, Start/Stop buttons, 6 status states (Listening / Thinking / Speaking / Connecting / Error / Idle), how-to guide, 3 example phrases. Main: chat transcript via st.chat_message. Stats bar: total turns ┬Ę you said ┬Ę JARVIS replied. Auto-refreshes every 0.7 s via time.sleep(0.7) + st.rerun().

STT

Groq Whisper Large v3

PCM bytes joined ŌåÆ WAV container built with io.BytesIO + wave (16 kHz, mono, Int16). Buffer given .name = "audio.wav" attribute for Groq SDK. Sent to groq.audio.transcriptions.create(model="whisper-large-v3", response_format="text"). Returns plain string. Errors caught + logged ŌåÆ returns "" on failure, loop continues.

LLM

Groq LLaMA 3.3 70B

llama-3.3-70b-versatile via groq.chat.completions.create(). Full messages[] list passed on every call. System prompt pinned at index 0 ŌĆö never trimmed. Bounded: system + last 40 messages (20 turns ├Ś 2). Hard params: max_tokens = 256, temperature = 0.7. Errors return "" ŌĆö turn skipped.

TTS

pyttsx3 SAPI5 ŌĆö Offline

Windows SAPI5 engine via pyttsx3. Rate 165 wpm, volume 1.0. Engine is re-created on every _speak() call (pyttsx3.init() inside the method ŌĆö not stored on the object). Fully offline, zero API cost. Output goes directly to Windows SAPI5 ŌĆö does NOT route through AudioHandler.output_queue.

Threading Model

3 Concurrent Threads

ŌæĀ Streamlit main ŌĆö reruns every 0.7 s, drains _transcript_q, renders UI. ŌæĪ Agent daemon ŌĆö private asyncio loop: VAD ŌåÆ STT ŌåÆ LLM ŌåÆ TTS ŌåÆ flush. Ōæó PyAudio capture ŌĆö always running, feeds input_queue. Bridge: module-level _transcript_q (Queue) + _status (dict + Lock) in app.py.

ŌÜĀ’ĖÅ Technical Notes & Design Gaps

Gap 1 ŌĆö Tools

GROQ_TOOLS Defined But Not Wired

core/tools.py exports GROQ_TOOLS in OpenAI-compatible JSON schema, but _ask_groq() in core/agent.py:229 does NOT pass tools=GROQ_TOOLS to the API call. book_demo, check_slot_availability, get_current_time are fully implemented ŌĆö but JARVIS currently answers tool queries from LLM knowledge only, not actual function calls.

Gap 2 ŌĆö Memory

ConversationMemory.get_history() Unused

memory.add_turn() IS called after each exchange (for logging/display). However memory.get_history() is never called by the agent. The actual LLM context lives in messages[] maintained directly in _main(). ConversationMemory is a parallel log ŌĆö not the LLM's history source.

Gap 3 ŌĆö pyttsx3

Engine Re-Init on Every Speak Call

pyttsx3.init() runs inside _speak() on every turn ŌĆö creating and destroying the SAPI5 engine each response. Works reliably but adds ~50ŌĆō100 ms overhead per turn. A persistent engine in __init__ would reduce latency; re-init avoids cross-turn COM state leakage on Windows.

Gap 4 ŌĆö AudioHandler

Legacy Output Stream / Playback Thread

AudioHandler opens a 24 kHz output stream and starts a playback thread consuming output_queue. Nothing writes to output_queue ŌĆö pyttsx3 speaks via SAPI5 directly. The output stream, playback thread, and interrupt() drain logic are legacy scaffolding from the initial design and have zero functional impact.

Gap 5 ŌĆö numpy

numpy Listed in requirements.txt But Unused

numpy appears in requirements.txt but is never imported in any source file. All PCM math is done via audioop.rms() (stdlib C-extension) and raw bytes operations. numpy can be removed from requirements without any functional impact.

Gap 6 ŌĆö VAD Constants

MIN_SPEECH_FRAMES & SILENCE_LIMIT Not in config.py

MIN_SPEECH_FRAMES = 5 and SILENCE_LIMIT = 25 are module-level constants defined directly in core/agent.py, not in config.py. To tune VAD timing you must edit agent.py directly ŌĆö they are not exposed alongside the other audio config values.

­¤Éø Critical Bug Fixes ŌĆö Root Cause Analysis

BUG 1 ŌĆö FIXED

Streamlit Thread Isolation Violation

Symptom: AttributeError: st.session_state has no attribute "transcript" ŌĆö agent crashed on every transcription.

Root cause: Streamlit's session_state is bound to the main script execution context (ScriptRunContext). Writing to it from a background daemon thread is undefined behavior ŌĆö it raises KeyError or AttributeError.

Fix: Introduced module-level _transcript_q (queue.Queue) and _status (dict + threading.Lock). Agent writes to these ŌĆö safe from any thread. Streamlit reads from them on every rerun via _drain_queue().

BUG 2 ŌĆö FIXED

Acoustic Echo Loop

Symptom: JARVIS was speaking continuously without listening ŌĆö it was having a conversation with itself.

Root cause: PyAudio's capture thread runs at all times, including while pyttsx3 is playing TTS through the speaker. The speaker audio was picked up by the microphone, fed back into input_queue, transcribed by Whisper, and replied to ŌĆö creating an infinite self-conversation loop.

Fix: _flush_mic_after_speaking() ŌĆö immediately after _speak() returns: sleep 400ms for echo to decay, then drain all accumulated chunks from input_queue before the next listen cycle begins.

BUG 3 ŌĆö FIXED

Noise Hallucination / Phantom Input

Symptom: JARVIS would start speaking without the user saying anything ŌĆö answering questions nobody asked.

Root cause: SILENCE_THRESHOLD=500 sat below Windows ambient noise floor (fan, AC hum, electrical noise typically 300ŌĆō700 RMS). A brief noise spike triggered speech_started=True with just 1 frame, then 1.6s of near-silence was sent to Whisper ŌĆö which hallucinated text ("Thank you.", "You", random phrases) from the noise.

Fix: Raised SILENCE_THRESHOLD to 800 RMS. Added MIN_SPEECH_FRAMES=5 gate ŌĆö requires at least 5 consecutive frames (Ōēł320ms) above threshold before treating input as real speech. Noise bursts are silently discarded.
